정렬 필터기능 완성

참고 report 초안 작성

발표자료준비
-사전 조사 내용: FIGMA 페이지, NOTION 프로젝트 설명 페이지, 실제 Youtube 동작참고
-프로젝트 요약: 프로젝트 목표/핵심요구사항(주요 결과, 성과)/사용자시나리오(핵심 기술, 접근 방식)/기대성과(프로젝트 의의)
-주요 개발 내용:조원별 주요 개발 기능 설명
상단바HTML 메인페이지-필터 html, 메인페이지-필터 CSS, 채널페이지 CSS, 메인페이지 JS(home.js), loadUI.js 등
고도화: 태그필터 구현, 추가 정렬 필터 버튼 구현 현재 재생영상관련 추천재생목록알고리즘 적용

-개발 소감
추천 재생목록을 구현하면서 429 http status code (연속호출 허용 범위 초과) 오류코드 등을 해결하고 어떻게 하면 API요청을 최소화할 수 있을까 고민한 것이 인상깊었습니다.

-개발 중 장애물 / 극복방법
HTML `data-*` 속성에 접근할 때 JavaScript의 `dataset` 객체는 속성 이름을 자동으로 카멜 케이스로 변환합니다.
이에 대한 자세한 내용은 data-속성-카멜-케이스-변환.md파일을 참고

또한 상단바의 검색창에 한글만 입력되고 다른 입력은 안되는 현상이 있었습니다.
이에 대한 자세한 내용은 input-요소-한글-문제-report.md파일을 참고

-추가/리서치 기능
사용자가 마지막으로 영상을 본 시점을 기억해두었다가 해당 영상에 다시 진입하면 그 시점부터 재생해주는 기능
지금은 영상이 29개로 적지만 나중에 무수히 많은 영상이 있을때 모든 영상을 한번에 로드할 수는 없으므로 사용자가 스크롤할수록 일정 수의 영상을 계속 로드할 수 있는 무한 스크롤 기능
